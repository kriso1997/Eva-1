# AutoMatch - Evaluación Front-End

Este proyecto es una landing page responsiva diseñada para la comparación de vehículos, desarrollada como parte de la evaluación de Front-End. Se ha priorizado la semántica web, la accesibilidad y el diseño moderno mediante el uso de CSS puro y utilidades de framework.

## 🚀 Tecnologías y Buenas Prácticas Implementadas

* **HTML5 Semántico (Rúbrica 1.1.1):** Uso estricto de etiquetas semánticas (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<nav>`) para mejorar el SEO y la accesibilidad de lectores de pantalla.
* **CSS3 Moderno (Rúbrica 1.1.3):** * Implementación de **Flexbox** para la alineación del Header y la distribución de las tarjetas de contenido.
  * Uso de **Variables CSS (`:root`)** para mantener una paleta de colores consistente (Dark & Violet Premium).
* **Accesibilidad (WCAG AA):**
  * Contraste de color verificado para legibilidad (texto claro sobre fondos oscuros).
  * Estados `:focus` claramente definidos con `outline` visible para navegación por teclado.
* **Diseño Responsivo:** Estructura fluida que se adapta a dispositivos móviles y de escritorio.

## 🤖 Documentación de Uso de IA (Rúbrica 1.1.2)

Durante el desarrollo de este proyecto, se utilizó Inteligencia Artificial (Gemini) como un asistente estratégico (Pair Programming). El proceso consistió en:
1. **Generación de estructura base:** Creación del esqueleto de cuadrículas (grids) y tarjetas de presentación.
2. **Refactorización y debugging:** Identificación de errores de anidamiento en el DOM (etiquetas mal cerradas) y corrección de la jerarquía visual.
3. **Optimización W3C:** Asegurar el uso de atributos `alt` en imágenes y roles semánticos (Aria-labels) en la navegación.
La IA no reemplazó la toma de decisiones, sino que actuó como un revisor de código riguroso para garantizar el cumplimiento de la rúbrica y las buenas prácticas de la industria.

## ⚙️ Control de Versiones (Rúbrica 1.1.4)

El proyecto fue gestionado mediante Git/GitHub, utilizando un historial limpio con **Commits Semánticos** (Conventional Commits) como `feat:`, `style:`, y `docs:` para facilitar la trazabilidad del desarrollo.
